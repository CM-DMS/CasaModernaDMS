**Key design rules baked into the editor:**
- `rate` is inc-VAT (inclusive tax template)
- NEVER compute `grand_total` / taxes in React — server-authoritative; full doc replaced on every save
- `cm_final_offer_inc_vat` is the price floor — below-floor triggers 🔒 warning
- `saveInProgressRef` mutex prevents double-saves
- Tax template rows fetched from ERPNext and sent explicitly on REST saves
- Sales Invoice conversions navigate to `/new` without auto-saving (user assigns name on explicit Save)
